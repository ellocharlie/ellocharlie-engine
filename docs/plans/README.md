## Adding a Plan

1. Write the plan as a Markdown file in `docs/plans/` with the naming convention `YYYY-MM-DD-slug.md`.
2. Add a row to the table above with the date, a linked title, and status (`Draft` / `Active` / `Phase N Complete` / `Complete` / `Superseded`).
3. Update status in this table as the plan progresses — do not edit completed plans in place; append a revision section.

## Plan Template

```markdown
# [Initiative] Implementation Plan

**Version:** 1.0
**Date:** YYYY-MM-DD
**Owner:** [name or agent codename]
**Status:** Draft | Active | Complete

---

## 1. Objective
## 2. Phases & Timeline
## 3. Success Criteria
## 4. Resources Required
## 5. Risks & Mitigations
## 6. Revision History
```
